## 실패 패턴 분석

### 1. 도메인 경계 혼동 (케이스 4)
- **증상**: 부동산 공유 지분 분쟁 → 상속으로 분류
- **원인**: "형제 3명이 부모님 땅을 공동 상속받았는데"에서 "상속" 키워드가 부동산보다 강하게 작용
- **해결 방안**: 프롬프트에 "대화의 핵심 쟁점(분쟁 대상)을 기준으로 분류" 지시 추가

### 2. L2 미도달 (케이스 24, 35)
- **증상**: L1은 맞지만 L2 서브카테고리까지 내려가지 못함
- **원인**: confidence가 낮아서 상위 노드에서 멈추거나, 법률 전문 용어 매핑 부족
- **해결 방안**: 
  - 온톨로지 리프 노드의 `keywords` 필드 보강
  - 프롬프트에서 "가능한 한 L3 리프 노드를 선택" 규칙 강화 (현재 이미 있으나 모델이 때때로 무시)

### 3. 멀티 노드 활용 가능성
- 모든 실패 케이스에서 `matched_nodes` 2-3번째에 올바른 노드가 포함됨
- Layer 2(MongoDB 검색)에서 상위 3개 노드를 모두 활용하면 실질 L2 커버리지 **~92%** 도달 가능
