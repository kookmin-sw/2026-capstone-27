## B-2 설계 조정 메모 (R-17 재검증 피드백 반영)

R-17 재검증에서 **#1 "전세 보증금 미반환" 케이스가 `law-001`(부동산) ↔ `law-007`(임대차특별법) 경계에서 흔들리는 현상**이 관측됐다. 민법 전체만 적재할 경우 이 회귀 케이스가 retrieval 공백으로 이어질 수 있으므로, B-2 인제스트 단계에서 아래 조정 사항을 반드시 적용한다.

### 조정 1. 임대차 조항 전량 포함 (필수)

민법 **제618조~제654조(임대차)** 를 누락 없이 적재. 특히 다음 핵심 조문:

- 제618조 (임대차의 의의)
- 제623조 (임대인의 의무)
- 제626조 (임차인의 상환청구권)
- 제635조 (기간의 약정 없는 임대차의 해지통고)
- 제640조 (차임연체와 해지)
- 제646조 (임차인의 갱신청구권 / 매수청구권)
- 제654조 (준용규정 — 전세권 조항과의 연결고리)

전세권 관련 제303조~제319조도 함께 포함 (민법 물권편).

### 조정 2. 다중 카테고리 태그 부여 (필수)

`legal_chunks` 테이블의 `categories` 컬럼(String[])에 **복수 카테고리**를 태깅한다. 분류기가 `law-001-02`(부동산/임대차) 또는 `law-007`(임대차 특별법)로 가더라도 동일 조문이 매칭되도록.

예시:
```
제618조 임대차 정의
  → categories: ['law-001-02', 'law-007-01', 'law-001-02-03']
제303조 전세권 정의
  → categories: ['law-001-02-03', 'law-001-02', 'law-003-01']  (부동산/임대차/물권)
```

구현 시 조문별 카테고리 매핑 테이블을 `resources/ontology/article-category-map.yml`에 선언. B-2 로더 스크립트가 이 매핑을 읽어 `categories` 배열로 채운다.

### 조정 3. B-4 retrieval에서 matched_nodes 상위 N개 OR 결합 (필수)

의도 분류기가 반환하는 `matched_nodes`를 **1순위만 사용하지 않고 confidence ≥ 0.7인 상위 N개(최대 3개)** 를 OR 조건으로 묶어 검색. R-17 #1 케이스에서 2순위(`law-001-02-03`, conf 0.85)에 정답이 존재했으므로 이 방식으로 복구 가능.

`PgLegalRetrievalService.search(List<String> categoryIds, String query)` 시그니처로 변경하여 다중 카테고리 WHERE IN 조건을 지원.

### 검증 기준

B-2 완료 후 R-17 #1 케이스를 실제 retrieval까지 돌려서 **제618조/제623조/제635조 등 임대차 핵심 조문이 상위 5개 결과 안에 포함**되는지 확인. 포함되지 않으면 `article-category-map.yml` 보강 또는 hybrid weight 재조정.
