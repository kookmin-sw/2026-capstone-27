## 변화 분석

### 순수 퇴보 케이스 (1건)

**#1 전세 보증금 미반환** — Cohere가 `law-007-01-05`(주거 임차 관련)로 라우팅.
Legacy LLM은 `law-001-02-03`(부동산 > 임대차 > 보증금 반환)을 선택했음.

- 입력 쿼리: "전세 계약 종료 후 집주인의 보증금 미반환"
- Cohere 1순위 `law-007-01-05` conf 0.9, 2순위 `law-001-02-03` conf 0.85 (2순위에 정답 존재)
- 원인 추정: 온톨로지에 "주거 임대차"가 `law-001-02`(부동산 하위)와 `law-007`(임대차 특별법 계열) 양쪽에 걸쳐 있어 경계 혼동
- 대응: 프롬프트에 "부동산 거래 자체가 쟁점일 땐 `law-001` 하위 우선" 라우팅 힌트 추가, 또는 `matched_nodes` 상위 2개를 retrieval에 활용

### L3 리프 개선 케이스 (3건)

`law-002-02`, `law-005-02`, `law-005-03`에서 Cohere는 **L3 리프까지** 정확히 도달한 반면 Legacy LLM은 L2에서 멈춤. 이들은 "전체 정확도" 집계에는 영향 없지만 Layer 2(RAG) 검색 범위를 좁히는 데 유리하다.

### 공통 실패 케이스 (3건)

#3 공유 지분 분쟁, #23 유류분 청구, #34 사용자 책임 — 두 모델 모두 실패. 온톨로지/프롬프트 레벨의 구조적 한계로 판단되며, 모델 교체로는 해결 불가.

- #3: 부동산/상속 경계 (두 모델 모두 상속 쪽으로 기울지만 하위 노드는 서로 다름)
- #23: 유류분 리프 노드(`law-003-04`)의 키워드 매칭이 약함
- #34: "동물 점유자 책임" 법률 개념이 쿼리에서 추출되지 않음

## 검증 결과: 조건부 PASS

**정확도 자체는 약간 하락(L1 −8.7%p, L2 −8.3%p)했으나**, 차이는 **샘플 크기 12의 통계적 노이즈 범위 안**이다(케이스당 ±8.3%p). 실질적 회귀는 #1 한 건이며, 이는 프롬프트 튜닝 또는 `matched_nodes` 상위 2개 활용으로 복구 가능.

### 권장사항

| 우선순위 | 항목 | 설명 |
|---|---|---|
| P1 | **현 상태 유지** | 분류 파이프라인을 `command-a-03-2025`로 그대로 운영. 단일 회귀 케이스(#1)는 차후 프롬프트 튜닝 또는 multi-node retrieval로 보정 |
| P2 | 프롬프트 라우팅 힌트 추가 | "거래/계약 자체가 쟁점일 땐 `law-001` 하위 우선" 류 규칙 추가 |
| P2 | 테스트 케이스 확장 | 12건은 과소. Phase B 검증 시 도메인별 5~10건으로 30~40건 수준 확장 권장 |
| P3 | `matched_nodes` 상위 N개 활용 | Layer 2 검색을 1순위 노드에만 의존하지 말고 conf ≥ 0.7 노드 전부 사용 |
| P3 | 지연 최적화 | 평균 7.5s는 사용자 대기 시간으로 긴 편. `command-r-08-2024`(더 작은 모델) 비교 측정 고려 |

### 모델 전환 결론

L1/L2 정확도 소폭 하락은 **허용 범위 내**. Cohere로의 전환을 **유지**하며, 향후 테스트 케이스 확장과 프롬프트 튜닝으로 격차 복구를 시도한다.
